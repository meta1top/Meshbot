## C. 已知做不到 / 不在范围（别浪费时间试）

- **观察者中断他人发起的 run**：设计上禁止（schema / 云端 / 设备侧三处独立校验），
  理由是「中断是破坏性的、无从仲裁」。本轮只是把按钮改成诚实的禁用态。
- **R2b：watch 注册与快照拉取无时序保证**。结构上存在裂缝（注册完成前到达的
  生命周期事件会被直接丢弃，那条会话要等下次重拉才出现），但**裂缝宽度未取证**，
  改动也大，本轮刻意未动。取证方案：客户端记录 mirror 帧的 `seq`，若某 agent 收到的
  首帧 `seq > 1`，即证明有帧落在注册前被丢弃。
- **云端孤儿 agent 行清理**：SQL 已交付
  （`apps/server-main/migrations/202607210942-cleanup-orphan-agent-rows.sql`），
  **需 DBA 手动执行**，先跑第一步只读查询确认会删什么。软删行会复活并保留云端
  agent id（该 id 在 URL 与 localStorage 里），所以只清「宿主设备已撤销/已不存在」
  这类不可能复活的。
- **web-main 的会话为远程只读**：没有改名 / 删除入口，相关双写风险天然不存在。
