# UI 精致化（紧凑专业型 · 标杆轮）实施 Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** 把 web-agent 提升到紧凑专业型（Linear/Notion 风），本轮先搭组件地基 + 做网盘/技能两个标杆页验证手感。

**Architecture:** 改 `packages/design` 共用组件（Button/Input/Card + 新增 Table）+ web-agent 标题栏统一 + 两个标杆页页面级精修。组件改动天然全局生效，标杆页在此之上做页面级精致。

**Tech Stack:** Tailwind CSS v4（无 config 文件，`@theme inline` + CSS 变量）、cva、shadcn new-york、lucide-react、Next.js。

## Global Constraints

- **紧凑刻度（用具体 px 值，不加全局 fontSize token）**：Tailwind v4 改 `--font-size-base` 会覆盖默认、波及全仓未改页面 —— **禁止**。组件/页面直接用具体值（项目本就大量 `text-[13px]` 写法）。字号约定：`11px` 辅助/meta、`12px` 次要/表头、`13px` 正文基准、`14px` 强调、`15px` 标题栏、`18px` 大标题（少用）。
- **控件高度**：`sm = h-7`(28px)、`default = h-8`(32px)。标题栏统一 44px。表格行 30px。
- **圆角**：控件/卡片 `rounded-md`，弹窗/大卡 `rounded-lg`。
- **组件改动全局生效**：改 Button/Input/Card/PageHeader 会让所有引用页随之变样——这是预期收益。其它非标杆页若出现轻微错位，记录留全量轮，**不在本轮逐一追**。
- **不动颜色语义**：只改密度/排版/尺寸，不改颜色 token、不改深浅主题。
- **验证靠真实渲染**：前端无完善自动测，typecheck + biome 必过；视觉靠用户在 app 真实渲染验收。纯逻辑（Table 排序）抽函数加单测。
