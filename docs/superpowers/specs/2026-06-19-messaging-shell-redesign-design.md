# 桌面端消息壳重构 + Slack 化精修（web-agent）

> 设计文档 · 2026-06-19 · 范围：`apps/web-agent`（桌面端 UI，纯前端结构与交互重构，不改后端业务逻辑）

## 背景与目标

当前桌面端有四个一级模块（主页 / 消息 / 助手 / 更多），消息与助手是两套独立的侧栏、路由、页面；右侧还有一个绑定当前 IM 对话、帮你起草回复的「Agent 建议」伴生面板。整体布局偏散，列表（频道 / 私信 / 会话）密度和层级不如 Slack 精致。

本次目标：**对齐 Slack 的信息密度与壳结构，把「消息 + 助手」收敛成一个统一的消息中心**，同时保留 meshbot 自己的暖炭灰 + 橙色视觉身份。

五条需求：

1. 精修频道列表、私信列表、对话内容列表的细节（密度、层级、对齐 Slack）。
2. 借鉴 Slack：顶栏搜索之后放一个**全局助手入口**，点击从右侧打开一个 **shell 级**面板。
3. 移除「助手」「主页」两个一级模块，合并进「消息」；首页即消息。最终一级导航只剩 **消息、更多**。
4. 消息侧栏分三区：**频道 / 私信 / 助手**（助手区 = 原来所有 Agent 会话）。
5. 统一「新消息」：无论发起频道 / 私信 / 助手对话，都从一处发起（参考 Slack 的「至：」）。

## 已确认的关键决策

- **移除旧伴生面板**：删除绑定 IM 对话、帮起草回复的「Agent 建议」面板（`im-companion-panel.tsx`），体验不佳。右侧面板今后**只**服务全局助手。
- **助手双层模型「随手问 + 沉淀会话」**：
  - **随手问** = 顶栏 ✦ 打开的全局助手面板，不绑定当前对话，可随处右侧浮出；
  - **沉淀会话** = 左栏「助手」区列出的命名会话（与频道 / 私信并列，点开主区全屏）；
  - 随手问的对话可「保存到助手」**沉淀**成左栏会话；**默认不进左栏，保存才进**，保持左栏只留用户在意的会话。
- **统计仪表盘去向**：原 `/assistant` 仪表盘（统计卡 + 26 周热力图）挪到「更多」作「使用情况」页；助手区未选会话时的着陆页 = 干净的「随手问」起点（输入框 + 建议 chips）。
- **输入框统一**：富文本（加粗 / 斜体 / 删除线 / 链接 / 有序·无序列表 / 行内代码·代码块）+ 附件（预览 chip 可删）+ 随内容自适应高度（到上限内部滚动）。频道 / 私信 / 助手 / 随手问**共用同一套输入组件**。
- **新消息单框分组**：一个「至：」框，下拉按 频道 / 成员（私信）/ 助手 三组（不用分段切换）。

## Shell 结构（最关键，反复确认过）

「**深色外壳 + inset 圆角内容卡**」，等同 Slack 现代壳，也是 meshbot 现有视觉语言：

```
┌────────────────────────────────────────────────────────┐  ← 窗口
│ ● ● ●   [   🔍 搜索 meshbot 通栏   ]        ✦  ?         │  ← 顶栏：深色外壳，无 border，全宽
├──┬─────────────────────────────────────────┬──gap──┬──┤
│导│ ╭─────────────────────────────────────╮ │ ╭────╮│边│
│航│ │ 侧栏(深) │ 主内容(白)               │ │ │随手││  │  ← 内容卡：侧栏+主区融合成一张统一圆角卡
│ra│ │ 频道     │  对话流 / 富文本输入      │ │ │问卡││  │  ← 随手问卡：右侧另一张同样的圆角卡
│il│ │ 私信     │                          │ │ │    ││  │  ← 两卡间一条同宽深色缝
│  │ │ 助手     │                          │ │ │    ││  │
│  │ ╰─────────────────────────────────────╯ │ ╰────╯│边│
├──┴──────────────────────────────窄深色边──────────────┤  ← 底部窄边
└────────────────────────────────────────────────────────┘
```

规则：

- **深色外壳** = 顶栏（顶，**无 border**，深色直接过渡到内容）+ 导航 rail（左，flush）+ **右侧与底部的窄深色边**。
- **内容卡** = 侧栏（深）与主内容（白）**融合成一张统一圆角卡**（`overflow:hidden` 裁切，外圈圆角），flush 贴 rail 与顶栏，右 / 下露窄边。
- **随手问卡** = 打开时在右侧作为**另一张同样的圆角卡**，与内容卡之间一条**同宽深色缝**；关闭时内容卡自动撑满到右窄边。
- 配色沿用 meshbot：外壳 / rail / 侧栏暖炭灰（`#1d1917`/`#241f1d`/`#2b2522`），主区白，强调橙 `#e8612c`。**不引入 Slack 紫**。

## 组件级设计

### 1. 一级导航 rail（`workspace-rail.tsx` / `rail-nav-item.tsx`）

- 导航项从 4 个缩到 **2 个**：消息（`💬`）、更多（`⋯`）。移除「主页」「助手」。
- 底部保留主题切换 + 用户头像菜单。
- `areaFromPath()`（在 `app-shell-layout.tsx` / `workspace-rail.tsx`）重写：`/`、`/messages*`、`/session*`、`/schedule*` 全部归入 `messages` 区；`/more*` 归 `more`。

### 2. 统一消息侧栏（新组件，替代 `im-sidebar.tsx` + `assistant-sidebar.tsx`）

- 头部：「消息」标题 + `✎` 统一新消息入口。
- 三个可折叠分段，统一的分段头样式（小号、muted、带 `+` 快捷新建 + 折叠 chevron）：
  - **频道**：`#` 公开 / `🔒` 私有；未读 = 加粗 + 橙色数字 pill。数据来自 `conversationsAtom`（type=channel）。
  - **私信**：在线状态绿点；未读 pill。数据来自 `conversationsAtom`（type=dm）+ `presenceAtom`。
  - **助手**：`✦` 图标；沿用现有 pinned / recent 排序与重命名 / 固定 / 删除菜单（`session-list-item.tsx`）。数据来自 `sessionsAtom`（仅 `pinnedToSidebar` 的会话，见数据层）。
- 行样式统一：高度 ~30px、`icon + name + 尾部 pill`、active 用 `accSoft` 背景、hover 浅底。
- 复用 `session-list-section.tsx` 的分段容器思路，三段统一为一个 `SidebarSection` 组件。

### 3. 主内容区（按会话类型多态渲染）

- 同一壳、同一主区位置，按当前选中会话**类型**选择渲染器：
  - 频道 / 私信 → IM 渲染（`im-message-list.tsx` 精修版 + `im-conversation-header.tsx`）。
  - 助手会话 → 会话渲染（`session/message-list.tsx`：markdown、tool calls、reasoning、token 用量，保留）。
- **对话区精修**（需求 1，对频道 / 私信 / 助手通用的展示层）：
  - **消息分组**：同一发送者连续消息只首条显示头像 + 名字，后续行 hover 才在左侧 gutter 显示时间。
  - **悬停操作条**：表情 / 回复 / 收藏 / 更多。
  - **表情回应 pill**、**日期分隔条**。

### 4. 共享富文本输入组件（升级 `common/chat-input.tsx`）

- 顶部格式工具条：B / I / U / 删除线 / 链接 / 有序·无序列表 / 行内代码 / 代码块。
- 附件：选择后在编辑区上方显示 chip 预览（图片缩略图 / 文档图标 + 文件名 + 大小 + 删除）。
- 自适应高度：随内容增高，到上限后内部滚动。
- 底部：附件 / 表情 / 提及 图标 + 橙色发送键；`Enter 发送 / Shift+Enter 换行`（随手问可 `⌘↩`）。
- 频道 / 私信 / 助手 / 随手问 / 新消息**全部复用**这一个组件（紧凑变体用于随手问面板）。

### 5. 统一「新消息」（新组件 + 主区视图）

- 点侧栏头部 `✎` → 主区进入「新消息」视图（**非弹窗**，与 Slack 一致）。
- 一个「至：」收件人框，下拉按三组：
  - **频道**：现有频道 + 「＋ 创建新频道…」。
  - **成员（私信）**：选人即发私信。
  - **助手**：「✦ 开启新助手会话」= 用下面正文直接发起一个 Agent 会话；也可挑已有会话继续。
- 正文用共享富文本输入。选定对象 + 写好正文 → 一次发出，路由到对应会话。

### 6. 随手问全局面板（shell 级，新组件，复用 `use-session-stream`）

- 顶栏 `✦` 高亮即开关（`shell-top-bar.tsx` 增加该按钮）。
- 作为 shell 级右侧卡（见 Shell 结构），**不绑定当前对话**，左侧内容卡仍可见可用。
- 面板头：品牌渐变头 + `✦ 随手问` + `🕘 历史 / ＋ 新对话 / × 关闭`。
- 「保存到助手」把当前随手问沉淀成左栏「助手」会话。
- **默认行为（待用户复审确认）**：点 ✦ 恢复上次随手问，`＋` 开新的；随手问默认不进左栏，保存才进。

## 数据层与路由

- **侧栏数据无需新模型**：统一侧栏同时读取 `atoms/im.ts`（`conversationsAtom` / `presenceAtom`）与 `atoms/sessions.ts`（`sessionsAtom` 派生 pinned / recent）。
- **随手问的「沉淀」语义**：需要区分「是否在左栏可见」与既有的「是否置顶」（`pinned` / `pinnedSessionsAtom` 是会话在左栏内部置顶，沿用不变）。提议在 Session 上**新增**一个独立标记 `inSidebar`（默认 false）：随手问新建的会话 `inSidebar=false`（只在面板历史可见），「保存到助手」置 true 才进左栏「助手」区。备选方案：随手问会话保持轻量 / 临时，「保存」时才 `createSession` 落库——但现有 `use-session-stream` / `createSession` / 历史基础设施已完备，**推荐前者（加标记）**更省事。具体字段与 SQLite + TypeORM 迁移在实施计划细化。
- **路由收敛**：`/` 重定向到消息中心；频道 / 私信沿用 `conversationId`、助手沿用 `sessionId`，两者 ID 空间不同，主区按类型选择渲染器。实施计划在「URL 统一 `/messages?c=<type>:<id>`」与「保留 `/messages` + `/session` 双路由、仅统一侧栏与壳」之间二选一（后者改动更小、风险更低，作为默认推荐）。
- `/assistant` 仪表盘逻辑迁至 `/more` 下的「使用情况」页；`/schedule`（定时任务）保留，入口从助手区移到对应位置（实施计划定）。

## 复审已确认（2026-06-19）

- **随手问默认行为**：点 ✦ 恢复上次随手问；`＋` 开新的；随手问默认不进左栏，「保存到助手」才进。
- **路由方案**：保留 `/messages`（频道 / 私信）+ `/session`（助手会话）双路由，仅统一侧栏与壳，主区按类型多态渲染（改动小、风险低）。不引入 `/messages?c=type:id` 统一 URL。
- **`/schedule` 定时任务入口**：沿用原助手侧栏底部位置——置于统一侧栏「助手」区底部的「定时任务」入口（改动最小）。
