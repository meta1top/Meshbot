# MeshBot 两栏壳重构（WorkBuddy 式）设计 spec

> **背景**：上一版统一 UI 重构（`2026-07-04-unified-ui-redesign-design.md`，P1–P5）已合入 main（PR #16）——把 web-agent / web-main 统一成**三栏 Slack 式共享壳**（深 rail + 浅子栏 + 常驻右区）。本 spec 是**方向性调整**：参考 WorkBuddy，转为**两栏 + 起手台**结构，同时保留我们已立起来的**暖炭·配橙**视觉身份。
>
> 本文是设计定稿（brainstorm 产物），非实施 plan。可视化 mockup（本地、gitignore）：`.superpowers/brainstorm/workbuddy-shell/mockup.html`（三帧：① 首页起手台 ② 会话态 ③ 右区两态）。

---

## 1. 动机

现状三栏壳信息密度高但"重"：一条深色窄 rail + 一条子栏 + 一条常驻右区，三列挤压中区正文，右区多数时间空置。WorkBuddy 走的是**两栏 + 起手台**：左一列导航、中一列正文，首页是一个"配置好一次任务再发起"的 composer 起手台。目标：

- **更宽、更清爽的正文区**（去掉常驻右列）。
- **任务优先的首页**：`/` 直接是起手台，降低"我该点哪"的冷启动。
- **保留 MeshBot 暖橙原创身份**，只借 WorkBuddy 的浅色两栏 + 留白结构，不抄它的冷灰绿。

## 2. 与 P1–P5 现状的关系（复用 vs 重做）

**复用**：暖炭·配橙 token（`--brand`/`--shell-*`/`--surface-0`）、Hanken 字体、共享包 `@meshbot/web-common`（源码直连子路径模式）、`SidebarSection`/`SidebarNavItem`/`BrandLogo` 等叶子、`ImMessageList`、`ChatInput`(TipTap)、`ArtifactPreviewPanel`、`AssistantDock`、per-session usage atom、两端 IntlProvider/主题/timeZone 基建。

**重做/改造**：
- 三栏 → 两栏：**深窄 rail + 独立子栏两列** 合并为**一条浅色宽侧栏**（方案 C）。
- **常驻右区拆掉**：产物→中区分栏；随手问/成员→右滑覆盖层。
- 新增 **`/` 起手台首页**。
- `PageShellView` 简化（侧栏移出内容壳）。

## 3. 目标信息架构（IA）

| 路由 | 内容 |
|---|---|
| `/` | **起手台**（新建任务落地页）：大标题 + 场景 preset + 模板 chips + 重 composer + 「最近」 |
| `/assistant` | 助手区（Agent 任务）——子栏列任务/设备 |
| `/messages`·`/messages/:id` | 消息区——子栏私聊/频道，中区会话 |
| `/skills` | 技能区 |
| `/drive`（文件） | 文件区 |
| `/flows` | 流程区 |
| `/settings/*` | 设置区（组织/设备/模型） |

一级 = 六区（助手/消息/技能/文件/流程/设置），二级 = 各区子项。**登录前**（P5 对话式设备授权 / 邮箱验证码）不变，沿用现状。

## 4. 布局：两栏

```
┌ 左栏(浅色·264px) ┐┌────────── 中区正文(白) ──────────┐
│ 品牌 MeshBot      ││                                  │
│ ⊕ 新建任务(→/)    ││   起手台 / 会话 / 区域内容        │
│ [一级图标条]      ││                                  │
│ ─────────────     ││   (产物预览时中区一分为二)        │
│ [二级宽子栏]      ││                                  │
│ 空间 · 用户       ││                                  │
└──────────────────┘└──────────────────────────────────┘
                        随手问/成员 = 从右滑入的覆盖层
```

去掉常驻右列。窗口窄时左栏可收成抽屉（响应式，后续阶段处理）。

## 5. 左栏（方案 C：顶部图标条 + 下方宽子栏）

一条浅色宽侧栏（~264px），一列装两级：

1. **顶部**：品牌 `MeshBot` + logo。
2. **「新建任务」CTA**（深底高亮）→ 跳 `/` 起手台。
3. **一级 = 图标条**：一排 6 个区域图标（助手/消息/技能/文件/流程/设置）+ 极小文字标签，当前区焦橙高亮，**始终可见、一键切区**。
4. **分隔线**。
5. **二级 = 宽子栏**：全宽显示**当前区**的子项（消息→私聊/频道两组；助手→任务/设备；设置→组织/设备/模型；home→「最近」任务）。复用 `SidebarSection` / `SidebarNavItem`（选中=白卡 `bg-(--shell-content) shadow-sm`）。
6. **底部**：空间(space)切换 + 用户头像/名 + 通知。

**与手风琴的区别**：一级不折叠、常显图标，切区只换下方子栏，比"展开一个区收起另一个"少一次操作。

## 6. 中区

### 6.1 首页起手台 `/`（最核心、最原创的一屏）

居中单列（max-w ~600px）：

- **Hero**：`MeshBot` + slogan（如"你的工作副驾"）。
- **场景 preset**（分段）：日常办公 / 代码开发 / 设计创意——切换预设的"技能+工具+模式"组合，改 composer 默认配置。
- **模板 chips**：文档处理 / 数据分析 / 写作润色 / 更多——点击往 composer 预填提示 + 选好技能。
- **重 composer**：`@ 引用文件，/ 调用技能` 占位 + 底部一排**发起前配置开关**（技能 / 连应用(MCP) / 权限）+ `+` / 发送。复用 `ChatInput`(TipTap)。
- **选择工作空间**。
- 发送 → 建新任务（session）→ 跳会话态（6.2）。
- 此态左栏子栏 = **「最近」**（最近任务）。

### 6.2 区域 / 会话内容

进入某会话（如 `/messages/:id`）：中区 = 会话头 + 消息流 + 底部 composer。其它区（技能/文件/流程/设置）= 各自内容页。均为两栏下的单一正文区。随手问由**右下角浮动气泡**全局唤起（见 §7），不再占会话头。

## 7. 右区落位（拆掉常驻右列）

| 原右区内容 | 新落位 |
|---|---|
| **产物预览** | **中区分栏**：出产物时中区一分为二（对话 \| 产物），常驻、可拖宽、可关。复用 `ArtifactPreviewPanel`，容器改为中区 split。 |
| **随手问** | **右下角浮动气泡（FAB）**：内容区右下角常驻一个小气泡，点击**展开成锚定右下的浮动面板**（可拉大 / 关回气泡），全局随时唤起、不占布局、跨区可用。复用 `AssistantDock`（chromeless）作面板内容。**取代**原顶栏 ✦ 触发 + 右滑覆盖方案。 |
| **频道成员** | **右滑覆盖层**：频道头点"成员"从右滑入弹层、可关。 |

删除持久右列相关 atoms/组件（`right-zone.ts` / `right-zone.tsx` / dock-tabs / 顶栏 ✦ 触发派生）。随手问气泡 = 一个 FAB 组件 + 展开态浮动面板（复用 `assistantPanelOpenAtom` 开关 + `AssistantDock`）；产物分栏与随手问气泡**互不遮挡**（气泡浮在最上层，产物在中区分栏内）。

## 8. 视觉语言

沿用 P1 暖炭·配橙 token，**不引入 WorkBuddy 冷灰绿**：
- 页面暖底 `--surface-0`(#faf7f2) / 侧栏浅暖 / 中区白 `--shell-content`。
- 焦橙 `--brand`/`--shell-accent`(#d24a0d)**克制**：只在选中态、主按钮、我方气泡、流式。
- 圆角 10–14px、1px 暖边、极浅阴影。Hanken 字体。暗色沿用暖炭·非纯灰。

**对比现状**：现状侧栏是**深 rail + 浅子栏**混搭；新版**整条侧栏转浅**（图标条 + 子栏同底），rail 不再是深色锚。

## 9. 措辞规范

- **品牌名**：用户可见文案统一 **`MeshBot`**（大驼峰）。实施时扫 i18n(zh/en) + wordmark 把历史 `meshbot` 改掉（代码标识符 / 包名 `@meshbot/*` / 路径不动）。
- **任务 vs 会话·消息**（落法 A）：
  - **「任务」**用于 **Agent 干活**那侧——起手台「新建任务」、助手区列任务、home「最近」任务、发起=创建任务。
  - **消息区**保持「私聊 / 频道 / 消息」——那是聊天，不叫任务。
  - 底层 `Session` entity 不改名，只改**用户可见口径**。

## 10. 组件级改造映射

| 现状（P1–P5） | 目标 |
|---|---|
| `workspace-rail.tsx`（深窄 rail 6 图标 + 主题/用户） | 拆：图标行→左栏顶部 `RailIconStrip`（浅、横向、6 区）；用户/空间→左栏底部；主题切换→设置或底部 |
| `messages/assistant/more/skills/drive-sidebar`（各区独立子栏） | 归入左栏「二级宽子栏」区，按当前区渲染；复用 `SidebarSection`/`SidebarNavItem` |
| `PageShellView`（内容壳含响应式抽屉侧栏） | 简化为纯内容壳（header + 滚动区）；侧栏移到外层持久 `WorkspaceSidebar` |
| `right-zone.ts` / `right-zone.tsx` / dock-tabs | 删除持久右列；产物→中区 split，随手问/成员→overlay |
| `(shell)/layout`（rail + 内容） | `WorkspaceSidebar` + 内容（两栏） |
| —（新增） | `/` 起手台页 + 场景 preset / 模板 chips / 重 composer 组件 |

新增顶层组件 `WorkspaceSidebar`（左栏容器：品牌 + 新建任务 CTA + `RailIconStrip` + `AreaSubList` + 底部），落 `@meshbot/web-common/shell`（两端共享），各 app 薄容器注入数据（atoms / react-query），延续 P3 容器/视图拆分范式。

## 11. 起手台元件语义（映射真能力）

| 元件 | 语义 |
|---|---|
| 场景 preset | 预设技能集 + 工具 + 模式组合，切 composer 默认配置 |
| 模板 chips | 预填提示 + 预选技能的任务模板 |
| composer「技能」 | 选本次任务启用的技能 |
| composer「连应用」 | 选本次连接的 MCP / 外部应用 |
| composer「权限」 | 本次任务的权限范围（默认/受限/…） |
| 发送 | 建新任务（session）+ 应用上述配置 + 跳会话态 |
| 「最近」子栏 | 最近的任务 / 会话 |

（具体给哪些场景/模板/技能是**产品内容**，spec 先放占位集，实施时可增删——不阻塞结构。）

## 12. 分阶段（每阶段独立可交付、可单独合）

- **Phase 1 — 两栏壳骨架**：`WorkspaceSidebar`（方案 C）替代 rail+子栏两列；`(shell)/layout` 改两栏、去右列占位；各区子栏迁入。先 web-agent。交付：能切区、各区内容照旧。
- **Phase 2 — 右区落位**：产物→中区分栏；随手问/成员→右滑覆盖层；删持久右区管线。交付：边聊边看产物、随手问/成员可滑出。
- **Phase 3 — 起手台首页**：`/` = 起手台（hero + 场景 + 模板 + 重 composer + 最近）；发送建任务跳会话。交付：首页起手台可发起任务。
- **Phase 4 — 两端拉齐 + 措辞 + 精修**：web-main 用上同壳（起手台可先占位）；MeshBot casing 扫 i18n/wordmark；任务/会话口径落实；暗色/响应式/精修。

每阶段沿用 subagent-driven-development（implementer→review→修→终审），走 PR 合 main。

## 13. 待定 / 延后（非阻塞）

- 起手台场景/模板/技能的**具体内容集**（产品内容，实施时定）。
- composer 发起前配置的**交互细节**（下拉 vs 弹层）——Phase 3 细化。
- 窄屏**响应式**（左栏抽屉化）——Phase 1/4 处理。
- web-main 起手台是否与 web-agent 完全一致，还是云端简化版——Phase 4 定。
