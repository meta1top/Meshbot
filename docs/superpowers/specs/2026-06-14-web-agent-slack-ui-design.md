# web-agent Slack 风格 UI 改造 · 设计稿

- 日期：2026-06-14
- 范围：`apps/web-agent`（前端），不涉及后端
- 状态：设计已与用户敲定，待 review → 进入实施 plan

## 1. 背景

meshbot 已从「单用户本地 Agent」演进为**协作型 Agent 平台**（本地 Agent + 云端团队协同）。本次参考 Slack 的视觉与信息架构，把 web-agent 的整体外壳与「助手」体验完全 Slack 化，并为未来的团队消息、全局主页、文件/知识库等能力预留导航位。

设计取向：**完全 Slack 化布局（最左 rail + 内嵌内容卡 + 顶部搜索栏）**，配色用 meshbot 的橙色而非 Slack 紫色。

## 2. 目标与非目标

### 本期目标
- 把 `AppShellLayout` 重构为 Slack 式外壳：最左 **rail** + 区域**侧栏** + **顶部搜索栏** + **内嵌白色内容卡**（四周橙色包边）。
- rail 作为一级导航，承载四大区：**主页 / 消息 / 助手 / 更多**。
- **助手区**完整实现（Slack 行式消息 + Agent 特有元素），承接现有全部 Agent 功能。
- **登录页**改为 Slack 风格（左品牌色块 + 右表单）。
- 主页 / 消息 / 更多三区落地**路由 + 占位骨架页**。
- 全程遵守 i18n（next-intl，禁止裸字符串）与现有 Form/useSchema 表单约定。

### 非目标（后续独立子项目）
- 消息区的**团队实时聊天后端**（频道/私信、WebSocket、消息持久化于云端轨）。
- 主页的**全局聚合数据**（跨设备未读、聚合统计）后端。
- 更多区的**文件 / 知识库**功能。
- 不改云端轨（server-main）、不动 Agent 编排逻辑。

## 3. 信息架构（rail 驱动四大区）

最左 rail 自上而下：工作区头像「M」→ 主页 → 消息 → 助手 → 更多 →（弹性留白）→ ☾ 主题切换 → 用户头像（含账号/组织菜单）。

| 区 | 图标位 | 内容 | 本期 |
|----|----|------|------|
| **主页** | ⌂ | 全局聚合：未读、本地会话入口、活跃统计/热力图（未来） | 占位骨架页 |
| **消息** | ✉ | 团队同事实时会话（频道/私信），对应 Slack 本体 | 占位骨架页 |
| **助手** | ✦ | 现有本地 Agent：会话列表 + Agent 对话 + 定时任务 | **完整实现** |
| **更多** | ⋯ | 文件、知识库等扩展入口 | 占位骨架页 |

- **rail 选中态**：高亮只作用于**图标方块**（半透明白 `rgba(255,255,255,.22)`），图标居中；下方文字标签**无背景色**，仅由暗→白提亮。
- **组织/账号设置**：从原侧栏顶部导航移到 rail 底部**用户头像菜单**（Slack 式），不再占一级导航位。
- **定时任务**：归入「助手」侧栏的一个分组（详见 §6），不再是一级导航。

## 4. 视觉设计系统

### 4.1 配色（已锁定）

| 角色 | 颜色 | 用途 |
|------|------|------|
| 骨架（chrome） | `#9a3412` | 顶部搜索栏 + rail + 内容卡四周包边，三者一体 |
| 侧栏 | `#d24a0d` | 区域侧栏背景（会话列表等） |
| 内容区 | `#ffffff`（暗色翻 `oklch(0.145 0 0)`） | 内嵌白色圆角内容卡 |
| 强调橙 | `#d24a0d` | 白底内的主按钮、链接、发送键、品牌 logo |
| 会话选中 | `rgba(255,255,255,.24)` | 侧栏内选中项的半透明白 pill |
| 工作区「M」 | 白底 + `#9a3412` 字 | rail 顶部工作区头像 |
| 用户头像 | 绿 `#16a34a` / 首字母色 | rail 底部 |

chrome 上的文字一律白/半透明白；白底内容沿用现有 `--foreground` 等中性 token。

### 4.2 圆角

引入 Slack 式圆角（内容卡、rail 图标方块、pill、按钮）。**不修改共享 `packages/design` 的 `--radius`（=0rem，web-main 仍在用）**，新圆角作为 web-agent 局部样式/局部 token 定义在 `apps/web-agent/src/app/globals.css`。

### 4.3 深色模式

- 保留亮/暗切换（rail 底部 ☾）。
- **橙色骨架 + 橙色侧栏在亮/暗两种模式都保留**（品牌外壳不变）。
- 仅**白色内容卡**在暗色模式翻为深色背景，内容区文字随之翻转。

### 4.4 主题 token 落点

- 新增 web-agent 局部 CSS 变量（如 `--shell-chrome`、`--shell-sidebar`、`--shell-radius`）定义在 `apps/web-agent/src/app/globals.css`，避免污染 `packages/design` 共享系统、避免影响 web-main。
- 强调橙等已有语义 token（primary/accent）可复用，必要时在 web-agent 局部覆盖为 `#d24a0d`。

## 5. 外壳结构（Shell）

```
┌──────────────────────────────────────────────┐  ← 顶部搜索栏（#9a3412）
│ ← →        🔍 搜索 Meta1                    ?  │
├──┬───────────┬───────────────────────────────┤
│  │           │ ┌───────────────────────────┐ │  ← 内容卡四周包边（#9a3412）
│ra│  侧栏      │ │  顶栏（标题/操作）          │ │
│il│ (#d24a0d) │ │───────────────────────────│ │
│  │           │ │  内容区（白）              │ │
│  │           │ │                           │ │
│  │           │ └───────────────────────────┘ │
└──┴───────────┴───────────────────────────────┘
```

- **顶部搜索栏**：全宽，置于 chrome 上。左：前进/后退（←→）；中：全局搜索框（本期 UI 占位，搜索逻辑后续）；右：帮助 `?`。
- **rail**：固定窄竖条，§3 的导航。
- **侧栏**：区域相关；助手区 = 会话列表（§6），其余区 = 占位。
- **内容卡**：白色圆角卡，四周留出 chrome 包边，左上角随侧栏拼接。卡内含「内容顶栏 + 主体」。

### 组件拆分（新增/改造）
- `WorkspaceRail`（新）：rail 容器 + 工作区头像 + 用户头像菜单 + 主题切换。
- `RailNavItem`（新）：图标方块 + 文字标签 + 选中态（替代部分 `SidebarNavItem` 职责）。
- `ShellTopBar`（新）：搜索栏 + 前进后退 + 帮助。
- `AppShellLayout`（改造）：组合 rail + 区域侧栏 slot + topbar + 内容卡；按当前区渲染对应侧栏与内容。
- 区域侧栏：`AssistantSidebar`（由现有侧栏会话列表逻辑重构而来）、`PlaceholderSidebar`（主页/消息/更多共用）。

## 6. 助手区（本期主体）

### 6.1 侧栏（`AssistantSidebar`）
- 头部：「助手」标题 + 新建会话按钮（✎）。
- 分组（可折叠）：**置顶** / **最近会话** / **定时任务**。
- 列表项：单行省略；选中项半透明白 pill；复用现有 `pinnedSessionsAtom` / `recentSessionsAtom` / 标题实时更新（WS `title_updated`）逻辑。
- 错误/加载/空态沿用现有 `SessionListSection` / `SessionListSkeleton` 行为。

### 6.2 内容顶栏
- ☆（置顶切换）+ 会话标题 + 右侧操作（重命名 / 导出 等，沿用现有会话操作）。

### 6.3 消息区（Slack 行式）
现有 Agent 元素映射到 Slack 行（头像 + 名字 + 时间戳 + 左对齐内容）：
- **用户消息**：绿色/首字母头像 + 「Grant」+ 时间戳 + 文本（沿用 `user-message-actions`）。
- **助手消息**：橙色 ✦ 头像 + 「助手」+ 时间戳 + Markdown（`markdown-content`，含列表/代码块/行内 code/流式光标 `.md-streaming`）。
- **工具调用块**（`tool-call-block`）：折叠卡片——⚙ 图标 + 工具名 + 目标 + ✓/运行中状态；展开看输入/输出。卡片样式适配白底圆角。
- **压缩提示**（`compaction-banner` / `compaction-row`）、**待发送**（`pending-list`）作为特殊行/分隔保留。
- 流式光标、消息分页锚定（`scrollContainerRef`）行为保留。

### 6.4 输入框（`chat-input` 改造为富文本风格外观）
- 顶部工具栏：B / I / U / 列表 / 代码 / 链接（**视觉为主**；本期不强制实现富文本编辑，纯文本输入即可，工具栏可先做装饰/渐进增强——实施 plan 再定）。
- 主体：多行输入，placeholder 随机（沿用现有逻辑）。
- 底部行：附件 ＋ / @ / emoji + **模型选择 chip**（如 `claude-opus-4-8`）+ 发送键（橙）。

### 6.5 助手默认/空态（无选中会话）
承接**现有首页内容**：问候标题 + 概览统计卡（指标 + 活跃热力图 `ActivityHeatmap`）+ 建议 chips（`SuggestionChips`）+ 新建会话输入框。即：现首页 `/` 的内容移入助手区默认视图。

> 备注：统计/热力图属于 Agent 使用数据，本期留在助手区；未来「主页」做全局聚合时可再迁移。

## 7. 占位区（主页 / 消息 / 更多）

- 各自有路由与 rail 入口。
- 侧栏 + 内容为统一的占位骨架（区名 + 「敬请期待」/ 简述未来能力的空态插画或文案），走 i18n。
- 不接任何后端。

## 8. 登录页（Slack 风格 · 左品牌右表单）

改造 `LoginPage` + `AuthShellLayout`：
- **左侧品牌色块**：橙色渐变（`#9a3412`→`#d24a0d`）+ logo（M + meshbot）+ 标语 + 装饰圆，右上角语言/主题切换。
- **右侧白底表单**：标题 + 副标题 + 邮箱 + 密码 + 「登录」（橙按钮）+ 「还没有账号？去注册」链接。
- **保留**现有 `loginSchema` + Form/FormItem/useSchema + `useLogin` 逻辑与错误提示，仅换布局/样式。
- 注册页 `/setup` 风格同步对齐（同一品牌色块语言）。
- 小屏：品牌块收起/置顶，表单为主。

## 9. 路由调整

| 区 | 路由（建议） | 说明 |
|----|------|------|
| 主页 | `/` | 占位（现首页内容迁入助手默认视图） |
| 消息 | `/messages` | 占位 |
| 助手 | `/assistant`（默认视图）+ 会话沿用 `/session?id=` | 现有 `/session` 路由保留 |
| 更多 | `/more` | 占位 |
| 定时 | 归入助手侧栏分组；`/schedule` 路由可保留 | |
| 组织/设置 | 移入用户头像菜单；`/settings/org` 路由保留 | |

> 路由命名为建议值，实施时可微调；核心是 rail 切换区、助手承接现有会话流。

## 10. i18n

- 所有新文案走 next-intl，新增/调整 `appShell`、`login`、新增 `messages`/`home`/`more`/`assistant` 等命名空间的 key，en + zh 同步。
- 遵守 `i18n-page` 技能：禁止裸字符串。

## 11. 响应式

- 桌面优先（Electron）。lg+ 显示 rail + 侧栏 + 内容卡。
- 小屏：侧栏可收起，rail 保留或转为底部/抽屉（实施 plan 细化）。
- 保留现有 Electron 标题栏安全区（`titlebar-safe` / mac 控件安全区）逻辑，适配新 chrome。

## 12. 测试

- 组件渲染/交互：rail 选中态、区切换、助手消息行（用户/助手/工具块/流式）、登录表单提交与错误态。
